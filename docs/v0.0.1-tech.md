# Mengdie v0.0.1 — as-shipped technical reference

This doc reflects the code state at the v0.0.1 ship commit (2026-05-10, tag
`v0.0.1`, branch `main`). It is companion to:

- `docs/v0.0.1-rebuild-plan.md` — the *plan* for v0.0.1 (Phase 0 thesis)
- `docs/releases/v0.0.1.md` — operator-facing release notes
- `docs/blueprint.md` — long-horizon engineering blueprint
- `docs/roadmap.md` — multi-version roadmap

Where this doc and the blueprint disagree, **this doc wins for v0.0.1**;
the blueprint is forward-looking.

---

## 1. Thesis recap (one paragraph)

Mengdie v0.0.1 is a minimum-viable AE-brain: an MCP server that persists
the propositional facts produced by AE pipeline artifacts
(`conclusion.md` / `review.md` / `plan.md` / `analysis.md` /
`retrospect.md`), exposes hybrid FTS5 + vector retrieval to host LLMs,
and runs an offline "dreaming" pass that clusters related facts and
asks an LLM to synthesize each cluster into a higher-level fact.
Per-fact validity intervals + supersession edges handle decision
evolution; a recall-counter + power-law decay handle staleness. The
v0.0.1 ship deliberately keeps the corpus narrow (AE artifacts only)
and the OSS surface minimal (+1 line in `Cargo.toml`: `sqlite-vec`).

## 2. Tech stack

| Concern | Choice | Rationale |
|---|---|---|
| Language | **Rust** | Strictest compiler guardrail for agent-written code; single binary; sub-5ms startup |
| Async runtime | `tokio` (full features) | rmcp + fastembed `spawn_blocking` integration |
| MCP SDK | **`rmcp` v1.3** (features: `server`, `macros`, `transport-io`) | Official Anthropic Rust MCP SDK |
| SQLite driver | **`rusqlite` v0.39** (features: `bundled`, `load_extension`) | Bundled SQLite (FTS5 included); `load_extension` enables sqlite-vec runtime registration |
| Vector store | **`sqlite-vec` v0.1.9** (adopted in F-006) | Replaces `vector.rs` brute-force scan; in-process, single DB file |
| Embeddings | **`fastembed` v5** (`all-MiniLM-L6-v2`, 384d) | Local ONNX Runtime; 2-10 ms inference; ~90 MB model cached at `~/.cache/fastembed/` |
| FS watcher | `notify` v8 + `notify-debouncer-mini` v0.7 | Library form only; no daemon ships in v0.0.1 |
| CLI | `clap` v4 (derive) | `mengdie` subcommand router |
| Logging | `tracing` + `tracing-subscriber` | All logs to **stderr** (stdout reserved for MCP transport) |
| Serialization | `serde` (+ `serde_yaml` for frontmatter, `serde_json` for MCP) | — |
| Config | `toml` v0.8 | `~/.mengdie/config.toml` |
| Time | `chrono` (serde feature) | RFC3339 strings in DB; `Utc` exclusively |
| LLM provider | **`ClaudeCliProvider`** (subprocess of `claude -p`) | Trait-abstracted (`LlmProvider`); credentials delegated to CLI; supports `--json-schema` flag (Path B) |
| Structured output | **claude-CLI `--json-schema`** (BL-027 Path B) | NOT `rig::Extractor` (Path A spike failed 2026-05-08); 0 Cargo dependency |

**Cargo.toml net delta** for v0.0.1 over v0.x: **+1 line** (`sqlite-vec`).

## 4. Core data structures

### 4.1 `MemoryEntry` (in-memory shape; mirrors `memory_entries` row)

```rust
pub struct MemoryEntry {
    pub id: String,                       // UUID v4
    pub project_id: String,               // git-inferred, e.g. "mengdie"
    pub source_file: String,              // relative path to the source markdown
    pub source_type: SourceType,          // enum: conclusion|review|plan|analysis|retrospect|synthesis
    pub knowledge_type: KnowledgeType,    // enum: factual|decisional|experiential
    pub title: String,
    pub content: String,
    pub entities: String,                 // comma-separated lowercase tags (NOT a normalized table)
    pub valid_from: DateTime<Utc>,
    pub valid_until: Option<DateTime<Utc>>,
    pub superseded_by: Option<String>,    // memory_entries.id of replacement
    pub recall_count: i64,
    pub avg_relevance: f64,               // EMA of relevance scores from search audits
    pub last_recalled: Option<DateTime<Utc>>,
    pub embedding: Option<Vec<u8>>,       // 384 × 4 bytes little-endian f32
    pub embedding_dim: Option<i32>,
    pub is_longterm: bool,                // promoted by dreaming pass
    pub created_at: DateTime<Utc>,
    pub content_hash: Option<String>,     // SHA-256 of canonical content; dedup key
    pub invalidation_reason: Option<String>,
    pub synthesis_cluster_hash: Option<String>, // only set for synthesis rows
}
```

### 4.2 `SynthesisInput` / `SynthesisDraft` / `SynthesisOutcome`

```rust
pub struct SynthesisInput<'a> {
    pub cluster_id: &'a str,
    pub source_facts: &'a [&'a MemoryEntry],
    pub instructions: &'a str,
}

pub struct SynthesisDraft {
    pub title: String,
    pub content: String,
    pub entities: Vec<String>,
    pub knowledge_type: KnowledgeType,
}

pub enum SynthesisOutcome {
    Synthesized(SynthesisDraft),
    Skipped { reason: String },
}
```

### 4.3 `LlmProvider` trait

```rust
pub type LlmFuture<'a> = Pin<Box<dyn Future<Output = Result<String, LlmError>> + Send + 'a>>;

pub trait LlmProvider: Send + Sync {
    fn complete<'a>(&'a self, prompt: &'a str, opts: &'a LlmOptions) -> LlmFuture<'a>;
    fn name(&self) -> &'static str;
}
```

The single concrete impl is `ClaudeCliProvider`, which `spawn`s
`claude -p` with a flag whitelist (`--model`, `--json-schema`, etc.) and
parses the wrapped JSON envelope. Subprocess stdout/stderr classified
by `ExitKind` + `IoOp` for structured error reporting.

## 5. SQLite schema

DB lives at `~/.mengdie/db.sqlite`. WAL mode enabled.
`PRAGMA foreign_keys = OFF` is the project-wide assumption (BL-015 /
BL-018 propose making this explicit at `Db::open` time).

### 5.1 `memory_entries` — the fact table

```sql
CREATE TABLE memory_entries (
    id                     TEXT PRIMARY KEY,        -- UUID v4
    project_id             TEXT NOT NULL,
    source_file            TEXT NOT NULL,
    source_type            TEXT NOT NULL,           -- CHECK via trigger
    knowledge_type         TEXT NOT NULL,
    title                  TEXT NOT NULL,
    content                TEXT NOT NULL,
    entities               TEXT NOT NULL DEFAULT '',
    valid_from             TEXT NOT NULL,           -- RFC3339 UTC
    valid_until            TEXT,
    superseded_by          TEXT,
    recall_count           INTEGER NOT NULL DEFAULT 0,
    avg_relevance          REAL    NOT NULL DEFAULT 0.0,
    last_recalled          TEXT,
    embedding              BLOB,                    -- 384 × 4 bytes LE f32
    embedding_dim          INTEGER,
    is_longterm            INTEGER NOT NULL DEFAULT 0,
    created_at             TEXT NOT NULL,
    content_hash           TEXT,
    invalidation_reason    TEXT,
    synthesis_cluster_hash TEXT
);
```

**source_type** allowed values (enforced by `BEFORE INSERT/UPDATE`
triggers `memory_entries_source_type_*_check`):
`conclusion | review | plan | analysis | retrospect | synthesis`.

**Indexes**:
- `idx_memory_project` on `(project_id)`
- `idx_memory_knowledge_type` on `(knowledge_type)`
- `idx_memory_entries_valid_until_id` on `(valid_until, id)` WHERE `valid_until IS NOT NULL` — used by supersession sweep
- `idx_memory_content_hash` UNIQUE on `(project_id, content_hash)` WHERE `source_type != 'synthesis'` — deduplication
- `idx_synthesis_cluster` UNIQUE on `(project_id, synthesis_cluster_hash)` WHERE `source_type = 'synthesis'` AND `synthesis_cluster_hash IS NOT NULL` — synthesis idempotency

### 5.2 `memory_fts` — FTS5 virtual table

```sql
CREATE VIRTUAL TABLE memory_fts
    USING fts5(
        title,
        content,
        entities,
        content=memory_entries,
        content_rowid=rowid
    );
```

Synchronized via three triggers on `memory_entries` (`memory_fts_insert`,
`memory_fts_delete`, `memory_fts_update`).

Tokenization: default FTS5 (unicode61 with diacritic folding); no
custom tokenizer. Multilingual quality at corpus scale is a deferred
question (see roadmap → v1.0 Tantivy trigger).

### 5.3 `vec_memories` — sqlite-vec virtual table

```sql
CREATE VIRTUAL TABLE vec_memories USING vec0(
    memory_id  TEXT PRIMARY KEY,
    embedding  float[384] distance_metric=cosine
);
```

Synchronized via three triggers on `memory_entries`
(`vec_memories_insert`, `vec_memories_update`, `vec_memories_delete`),
gated on `embedding IS NOT NULL AND embedding_dim = 384`.

Backing structures (managed by sqlite-vec extension):
`vec_memories_chunks`, `vec_memories_rowids`,
`vec_memories_vector_chunks00`, `vec_memories_info`.

**Known caveat** (BL-022): synthesis rows are written with
`embedding = NULL` because the synthesis pipeline does not embed its
own output. They are searchable via FTS5 only until a re-embed pass
runs.

### 5.4 `memory_synthesis_links` — fact-blame edges

```sql
CREATE TABLE memory_synthesis_links (
    source_memory_id    TEXT NOT NULL,
    synthesis_memory_id TEXT NOT NULL,
    created_at          TEXT NOT NULL,
    PRIMARY KEY (source_memory_id, synthesis_memory_id),
    FOREIGN KEY (source_memory_id)    REFERENCES memory_entries(id),
    FOREIGN KEY (synthesis_memory_id) REFERENCES memory_entries(id)
);
CREATE INDEX idx_syn_link_source    ON memory_synthesis_links(source_memory_id);
CREATE INDEX idx_syn_link_synthesis ON memory_synthesis_links(synthesis_memory_id);
```

The **only** general-purpose edge table in v0.0.1. Maps each synthesis
fact to the source facts it consolidates.

### 5.5 `memory_search_audit` + `audit_returned_facts` — F-002 audit link table

```sql
CREATE TABLE memory_search_audit (
    id          INTEGER PRIMARY KEY,
    query       TEXT NOT NULL,
    scope       TEXT,
    took_ms     INTEGER NOT NULL,
    searched_at TEXT NOT NULL
);
CREATE INDEX idx_memory_search_audit_searched_id
    ON memory_search_audit(searched_at, id);

CREATE TABLE audit_returned_facts (
    audit_id INTEGER NOT NULL,
    fact_id  TEXT NOT NULL,
    rank     INTEGER NOT NULL,
    PRIMARY KEY (audit_id, fact_id),
    FOREIGN KEY (audit_id) REFERENCES memory_search_audit(id),
    FOREIGN KEY (fact_id)  REFERENCES memory_entries(id)
);
CREATE INDEX idx_audit_returned_facts_fact_audit
    ON audit_returned_facts(fact_id, audit_id);
```

Records every `memory_search` call with timing + scope, plus the
per-fact ranked result set. Powers `mengdie audit-stats` (F-005).

### 5.6 `metrics` — global counters

```sql
CREATE TABLE metrics (
    key         TEXT PRIMARY KEY,
    value_int   INTEGER NOT NULL DEFAULT 0,
    value_float REAL    NOT NULL DEFAULT 0.0,
    updated_at  TEXT NOT NULL
);
```

Known keys: `search_count`, `search_nonempty_count`, `ingest_count`,
`conflict_count`, `audit_write_failures` (BL-017 wants this surfaced
in `mengdie stats`).

## 6. Data flow

### 6.1 Ingest

```
markdown file
  └→ parser.rs::parse_frontmatter     YAML frontmatter → MemoryFrontmatter
  └→ parser.rs::extract_entities      tags + headings → comma-separated entities
  └→ embeddings.rs::embed_with_meta   metadata-in-chunk encoding → 384-d f32
  └→ db.rs::insert_memory             content_hash dedup, INSERT memory_entries
       └→ trigger memory_fts_insert        synchronously updates FTS5
       └→ trigger vec_memories_insert      synchronously updates sqlite-vec
  └→ contradiction.rs::check_overlap  entity-tag overlap + valid_from comparison
       └→ if match: db.rs::supersede   sets valid_until + superseded_by + invalidation_reason
  └→ metrics::bump("ingest_count")
```

Triggered by:
- MCP `memory_ingest` (push from AE skill — primary path)
- CLI `mengdie ingest <file>` (manual)
- CLI `mengdie import <dir>` (cold-start batch)
- `watcher.rs::start_watcher` library (host-orchestrated daemon — opt-in)

### 6.2 Search

```
SearchParams {query, limit, scope, project_id, min_score}
  └→ search.rs::search_fts5            BM25-ish over title+content+entities
  └→ search.rs::search_vector          sqlite-vec cosine top-K
  └→ search.rs::rrf_merge              Reciprocal Rank Fusion (k=60 default)
  └→ search.rs::apply_boost_and_decay  is_longterm boost + decay-aware effective relevance
  └→ search.rs::filter_min_score
  └→ db.rs::hydrate                    fetch full MemoryEntry rows for top-N IDs
  └→ db.rs::bump_recall_count          increments recall_count, updates EMA(avg_relevance), last_recalled
  └→ db.rs::write_audit                memory_search_audit + audit_returned_facts
  └→ metrics::bump("search_count")
```

Audit row is recorded **after** hydration but **before** returning; on
write failure, `audit_write_failures` is bumped and the search still
returns successfully (best-effort audit).

### 6.3 Dream (offline reflection pass)

```
mengdie dream [--synthesize]
  └→ dreaming.rs::run_promotion_pass
       └→ for each candidate fact:
            effective = decay::effective_relevance(avg, last_recalled, now)
            if recall_count >= 3 AND effective >= 0.45 → SET is_longterm = 1
            if effective < decay::DEMOTION_FLOOR (0.20) → demote
  └→ if --synthesize:
       └→ clustering.rs::seed_neighborhood_clusters
            cosine-similarity-driven clusters; min size 2, max size 5
       └→ for each cluster:
            synthesis.rs::build_prompt           cluster-aware prompt
            llm.rs::complete(--json-schema)      via ClaudeCliProvider
            synthesis.rs::parse_synthesis_response
            db.rs::insert_synthesis              source_type=synthesis, embedding=NULL (BL-022)
            db.rs::insert_synthesis_links        fan-out links to source facts
```

Trigger model: **on-demand by default** (operator runs `mengdie dream`).
`com.mengdie.dream.plist` is a launchd template for opt-in cron mode.
`ReflectionTrigger` trait is **not implemented in v0.0.1** —
deferred per discussion 027 Topic 2 (operator decision; trait was
discussed but not built).

### 6.4 Decay (read-time derivation)

```rust
const HALF_LIFE_DAYS: f64 = 60.0;
const DEMOTION_FLOOR: f64 = 0.20;

decay_factor(days) = 2^(-days / HALF_LIFE_DAYS)
effective_relevance(avg, last_recalled, now) = avg × decay_factor(now - last_recalled)
should_demote(effective) = effective < DEMOTION_FLOOR
```

`avg_relevance` in DB is **never mutated by decay**; decay is applied
at read time wherever ranking happens (`search.rs::apply_boost_and_decay`
and `dreaming.rs` promotion checks). This keeps recall history
auditable.

### 6.5 Supersession (write-time consistency)

When `contradiction.rs::check_overlap` flags a new fact `B` as
contradicting an existing fact `A`:

```sql
UPDATE memory_entries
   SET valid_until         = B.valid_from,
       superseded_by       = B.id,
       invalidation_reason = 'superseded'
 WHERE id = A.id;
```

The supersession decision uses entity-tag set overlap + `valid_from`
ordering. The protocol is **not formalized as a contract document**
(BL-029 wants this).

## 7. MCP API

Server binary: `mengdie-mcp` (stdio transport).

Three tools, all accepting `project_id` (override) and `scope`
(`project | global`) parameters:

| Tool | Params | Returns |
|---|---|---|
| `memory_search` | `query, limit?, min_score?, scope?, project_id?` | `Vec<SearchResult>` (id, title, snippet=200ch, score, source_file, knowledge_type, valid_from, recall_count) |
| `memory_ingest` | `source_file, project_id?` | `IngestResult` (memory_id, was_duplicate, contradicted_ids) |
| `memory_invalidate` | `memory_id, reason, superseded_by?` | `InvalidateResult` (memory_id, valid_until) |

Registered in Claude Code's `~/.claude/settings.json` as an MCP server.

## 10. Operational notes

### 10.1 Hot paths

- MCP `memory_search`: p50 ~20 ms on the dogfood corpus (69 facts, 5
  days × 21 calls). The audited average is **20.8 ms**. Includes
  FTS5 + vector + RRF + hydration + audit write.
- Ingest: ~30–80 ms dominated by fastembed inference (`spawn_blocking`).
- Dream synthesis: bound by Claude CLI latency (seconds per cluster).

### 10.2 Where state lives that can't be reconstructed from sources

If `~/.mengdie/db.sqlite` is deleted, you lose:

- All `recall_count` / `avg_relevance` / `last_recalled` history
- All `valid_until` / `superseded_by` edges (contradictions are
  re-derived on next ingest, but the original supersession timestamp
  is gone)
- All `memory_search_audit` + `audit_returned_facts` rows
- All `memory_synthesis_links` (synthesis facts themselves can be
  re-generated but the lineage to source facts is lost)
- The `metrics` counters

The source markdown files survive. `mengdie import <dir>` can rebuild
the fact table but cannot rebuild the above derived state.

### 10.3 What's *not* in v0.0.1 (deferred)

- **Reflection trigger framework** (`ReflectionTrigger` trait) —
  on-demand is the only trigger; no salience / composite / debounced
- **Memory lint / maintenance pass** — orphan GC, unresolved
  contradiction queue, embedding version drift checks (see
  discussion 016)
- **Entity as normalized node** — entities are a comma-separated TEXT
  field on `memory_entries`; no `entities` table, no `fact_entity`
  link table
- **Synthesis row re-embed** — synthesis rows have `embedding = NULL`
  (BL-022)
- **Daemon mode for watcher** — library only; orchestration is the
  host's responsibility
- **Cross-project search ranking** — `scope: global` works but does
  not weight by project relevance
- **MCP resources primitive** — only `tools` exposed in v0.0.1
- **Multi-host verification** — only tested in Claude Code; Continue
  / Cline / Cursor verification is a vNext task

### 10.4 Known correctness caveats (filed as BLs)

- **BL-022** synthesis rows ship with `embedding = NULL`; vector
  search misses them until re-embedded
- **BL-016** supersession query uses `JULIANDAY()` filter that is
  non-sargable; will hot at corpus scale
- **BL-023** `project_id` is inferred once at MCP server startup;
  stale if operator switches projects without restart
- **BL-037** `vec_memories` triggers don't downgrade-delete when a
  384-d row is updated to non-384 / NULL embedding
- **BL-013** `audit_returned_facts` can accumulate orphan rows on
  `rename_project` collision-merge
