# v0.0.1 PRD — Minimum-viable AE-brain

This is the **first PRD instance** under `docs/blueprint.md` §13
per-milestone PRD convention (filed as F-004 follow-up work, 2026-05-08).
Per §13 hard cap: ≤500 words per section.

## Goal

Ship the minimum-viable AE-brain that lets the operator stop re-doing
research already concluded, **without re-inventing wheels OSS already
provides**. Concretely: AE pipeline outputs (`conclusion.md` /
`review.md` / `plan.md` / `retrospect.md`) flow into mengdie via
push-primary ingestion; `/ae:analyze` reads back prior decisions as
Round-0 context injection; loop closes empirically (operator observes
fewer "didn't we settle this?" moments per session).

Traces to **blueprint §1** (system identity) + **blueprint §5 P0**
(four core promise items: AE artifact ingest path, `memory_search`
with provenance, AE plugin Round-0 injection, audit instrumentation)
+ **roadmap.md v0.0.1** (narrow OSS-adoption scope: keep all working
in-house code, swap 2-3 specific surfaces where OSS prevents
reinvention).

The narrow OSS-adoption scope is per CLAUDE.md FINAL section
(operator clarification 2026-05-05): mengdie is **AE 的大脑** — first,
AE-plugin specific; post-v1.0 generic possibility deferred to vNext+
dogfood-driven decision.

## Acceptance criteria

7 verifiable conditions, mixing bash spot-checks + operator sign-off:

1. **BL-026 sqlite-vec disposition resolved** — spike runs (PASS or
   FAIL); if PASS, sqlite-vec replaces `src/core/vector.rs` 264-LoC
   full-table-scan; if FAIL, sqlite-vec stays deferred + retry-trigger
   BL filed. Either outcome is acceptable; *resolution* is the AC, not
   adoption.
2. **BL-027 rig::Extractor disposition resolved** — same pattern:
   spike runs; if PASS, rig::Extractor replaces `src/core/synthesis.rs`
   ~100-LoC brace-depth JSON parser; if CONTINGENT/FAIL, retry-trigger
   BL filed. Disposition recorded, not adoption.
3. **BL-008 AE plugin Round-0 injection wired + tested** —
   `/ae:analyze` calls `memory_search` after Round 1 reviewer findings,
   before TL synthesis. Test: 3 consecutive `/ae:analyze` runs across 3
   different feature scopes inject Round-0 block with provenance.
4. **BL-025 AE plugin per-skill ingestion wired + tested** —
   `/ae:analyze`, `/ae:discuss`, `/ae:plan`, `/ae:review`, `/ae:work`,
   `/ae:retrospect` each call `memory_ingest` post-completion when
   appropriate `source_type`. Test: ≥3 of 6 skills produce ingestion
   in dogfood within 1 week.
5. **Production run reproduces 13-14 synthesis baseline** —
   `mengdie dream --synthesize` against operator's actual project DB
   produces ≥10 synthesis rows (matching empirical pre-v0.0.1 baseline
   per CLAUDE.md "13-14 syntheses produced empirically").
6. **All `cargo test` suites green** — 292+ tests pass post-
   integration. No regression from current state (v0.x final commit
   tested 292 passed / 5 ignored / 9 suites / 2.58s).
7. **MCP server registers + handles 3 tools end-to-end** —
   `mengdie-mcp` binary registered in `~/.claude/settings.json`;
   Claude Code invokes `memory_search` / `memory_ingest` /
   `memory_invalidate` round-trip without protocol errors.

## Out-of-scope

Explicit non-deliverables for v0.0.1 (deferred to vNext+ or v1.0+):

- **Multi-host adoption** (Continue / Cline / Cursor / OpenHands MCP
  registration). vNext task per roadmap.md.
- **Bi-temporal validity upgrade** (event_time + ingest_time per
  Graphiti pattern). v1.0 candidate, trigger = contradiction.rs
  false-positive empirically observed.
- **`ReflectionTrigger` trait alternative implementations** (salience
  / composite / debounced). vNext, BL-024.
- **mem0 entity-linking prompt borrow**. vNext if v0.0.1 dogfood
  reveals propositional-fact extraction insufficient.
- **Knowledge graph backend** (Kuzu / Neo4j / Graphiti integration).
  v1.0 via Graphiti adoption (reverse-leverage rule, not self-built).
- **Generic memory MCP server** (multi-tool ecosystem audience).
  vNext+ dogfood-driven decision; v0.0.1 strictly AE-plugin first.
- **Daemon-class infrastructure** (BL-010 deleted from 005-phase2-
  roadmap per F-004). `resources/com.mengdie.dream.plist` launchd
  covers periodic dreaming; daemon-class is mem0/Letta scope.
- **LLM-driven RAG search inside mengdie** (BL-012 deleted per F-004).
  Host LLM (Claude Code) is the RAG layer; mengdie returns top-N for
  host to synthesize.
- **Feedback / RL infrastructure** (BL-014 deleted per F-004).
  1-operator dogfood + manual roadmap adjustments suffice.
