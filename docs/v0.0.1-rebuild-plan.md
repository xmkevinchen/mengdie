# mengdie redesign — v0.0.1 migration outline

This is a **migration outline**, not a design doc. Each step is an
anchor; details for that step are developed via `/ae:analyze`,
`/ae:discuss`, `/ae:plan` when its turn comes — not pre-specified
here.

This is **not** v0.x → v1.0 incremental upgrade. v0.x design thesis
is being discarded. Version number resets to v0.0.1 to reflect a
fresh-start mengdie, not a successor numbered after v0.8.0.

## Why redesign

(Brief — full chat history is in commits and CLAUDE.md "Project
Status".)

mengdie's v0.x reinvented mature Rust infrastructure (swiftide RAG
framework, rig LLM agent framework, Qdrant / LanceDB vector stores)
and was scaled for "production data is precious" assumptions that
don't apply (data is reproducible from `docs/`). Solo project — can't
build everything from scratch. Karpathy LLM-wiki style is more
appropriate for the in-session LLM-driven processing layer (which
belongs in AE plugin, not mengdie). Reframe driven by user chat
2026-04-27.

## v0.0.1 thesis

1. **mengdie = AE 的大脑** — serves AE plugin first; generic ingestion
   from other sources is post-v0.0.1.
2. **mengdie 自成长** — cross-project meta-fact reflection. Knowledge
   becomes systematic via reflection over accumulated facts, not
   passive retrieval alone.
3. **开源 deferred** — v0.0.1 is personal-use rebuild, not yet
   accountable to external users.

| **F** | 版本号 cut + tag v0.0.1 + main migration (rename old `main` to `archive/v0.x`, this branch becomes new `main`). | manual git ops | E done |

**Implementation strategy**: incremental refactor on
`feature/v0.0.1-rebuild` branch. Each Step E BL keeps `cargo build`
+ `cargo test` passing. No from-scratch big-bang rewrite. Some BLs
(e.g., introducing swiftide / rig / Qdrant / LanceDB to replace
in-house ingestion or vector code) are larger chunks but still
incremental — feature flag / dual code path / gradual migration, not
"delete src/ and start over."

Old v0.x mengdie continues to run on the `main` branch for operator
use (e.g., reading `~/.mengdie/db.sqlite` via existing CLI subcommands)
through the entire v0.0.1 refactor. Operator workflow is not
disrupted until Step F.
