## Track 4 — Per-commit Doodlestein
**Strategic**: The `test_audit_stats_where_clause_shared` test catches string-level drift between the two SQL consts but does not prevent the deeper failure mode: a future refactor that deduplicates the WHERE clause into a Rust const or a SQL CTE would make both string assertions vacuously true while the actual query behavior diverges. The stronger form is a behavioral cross-check — run `AUDIT_STATS_SUPERSESSION_COUNT_SQL` against the same fixture as `test_supersession_sql_returns_expected_rows` and assert that its scalar result equals the sum of all `supersession_count` column values from `F002_SUPERSESSION_SQL`. That would be a true semantic invariant, not a substring membership check.
**Adversarial**: `test_audit_stats_supersession_count` calls `chrono::Utc::now()` twice in the same test to construct `now` and `valid_until`. Both calls are in the same lexical scope and almost always land in the same second, but under heavy load (e.g., a shared CI runner) the two `to_rfc3339()` strings can differ. More critically: `test_audit_stats_populated` does NOT assert `supersession_count_30d` — the AC1 description says "six-field snapshot," and `audit_stats_empty_db` asserts all six, but `test_audit_stats_populated` asserts only five, silently leaving the sixth field unverified for the non-empty case. **FIXED** in this commit by adding `assert_eq!(s.supersession_count_30d, 0)` to `test_audit_stats_populated` (also independently flagged by Codex Track 2 review).
**Regret**: The `fresh_db()` bootstrap in `tests/audit_stats.rs` — spawning `mengdie stats` to run migrations before each test — is the most fragile part of this change. It ties every integration test to the binary compile state, and if the `stats` subcommand changes behavior or exit code semantics, `fresh_db()` breaks silently. The actual regret risk is `fresh_db()` being cargo-culted into future integration tests without the sqlite-vec rationale, slowly coupling every new integration test to full binary compilation latency. This is the part most likely to get reworked when someone hits a slow CI cycle or a `mengdie stats` API change.
