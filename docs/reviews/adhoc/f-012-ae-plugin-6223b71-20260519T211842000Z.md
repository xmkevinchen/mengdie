## Disagreement Value Assessment

Both were looking at the same evidence — `source_type: plan` semantic in the protocol table maps to `/ae:plan`-document-content, not `/ae:work` execution outcomes.

TL synthesis followed challenger: the stronger framing (type confusion corrupts retrieval queries that filter by source_type) outweighs the weaker "acceptable" framing (codex saw it as not-explicitly-forbidden). The downstream cost of mixing plan-creation rationale with plan-execution outcomes in source_type=plan retrievals is real; the upstream cost of changing to `review` is zero. The asymmetry resolves to "fix it."

Pattern note: when cross-family acceptable vs adversarial BLOCK, lean adversarial when the cost asymmetry favors the fix.
