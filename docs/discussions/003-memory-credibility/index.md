When multiple users contribute to a shared team memory, individual brain quality varies. This directly affects team memory reliability.

## Problem Statement

Different users produce different quality knowledge. Some run full AE pipelines (analyze → discuss → plan → work → review), others skip steps. Some actively manage contradiction flags, others dismiss them. Some search frequently (giving Dreaming rich signal), others rarely do. If team memory promotion is purely democratic ("3 people have similar memories → promote"), quantity beats quality.
